时间字段与活动状态

抓取日期 · fetched_at系统首次抓到该内容的时间，首次入库后保持不变；历史缺失值使用已有时间作近似补齐。
更新日期 · update_time系统检测到正文内容发生变化的时间。首次抓取不算更新；内容未变时保持原值，未发生过变更可为空。
发布日期 · post_time来源平台公布文章或活动页面的时间；与系统何时发现它无关，源端缺失时以首次抓取时间近似。
Listing 生效时间effective_time_from_article，由程序从上下币文章正文确定性识别，通常表示开放交易、上线或下架生效时间，不是 AI 推测。
活动状态采用独立生命周期口径：开始时间晚于批次日为 Upcoming；开始时间不晚于批次日且结束时间不早于批次日为 Ongoing；结束时间早于批次日为 Ended。New / Updated 表示本次采集变化，不会覆盖生命周期，因此一条活动可以同时是 New 和 Ongoing。

ZMX 差异标签

未检索到同类未在 ZMX 基线内容中形成匹配。
已有同类 · 机制不同已有可比案例，但奖励、门槛或规则有实质差异。
已有同类型 · 粗粒度只确认大类做过，实际内容可能存在差异。
未进行对比对应类型不进行比较分析。

高 / 中 / 低由固定评分公式加总，不由模型直接定档。
事件类型（最高 30）＋ ZMX 差异（最高 30）＋业务影响（最高 20）＋原文与抽取置信度（−10～10）＋新颖度（最高 6）＋紧迫度（最高 6）。总分 ≥70 为高，40～69 为中，＜40 为低。
P1 → P7由内容类别和变化类型直接归档。
P1 新活动；P2 活动规则变化；P3 奖励变化及其他活动更新；P4 新产品；P5 产品更新；P6 上币或下架；P7 其他公告及发布早于批次 30 天以上的历史补录。同级按发布时间倒序，Highlights 每个平台最多 2 条，最终取前 5 条。
  • 趋势统计的是系统发现量，不等于官方当天发文量。
  • 地区独占首先是公告覆盖事实，不等于功能权限独占。
  • 最终决策仍需复核原文及业务约束。
